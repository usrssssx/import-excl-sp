# Acceptance Checklist (MVP)

Документ для приёмки проекта «Загрузка данных из Excel в смарт-процессы Bitrix24».

## 1. Матрица требований

| Требование | Статус | Где реализовано |
|---|---|---|
| Local app (iframe + внешний backend) | Реализовано | `routes/web.php`, `app/Http/Middleware/ResolveBitrixContext.php` |
| Установка приложения через install handler | Реализовано | `GET/POST /bitrix/local/install` |
| Сохранение контекста портала и пользователя | Реализовано | `ResolveBitrixContext::persistContext` |
| Список смарт-процессов | Реализовано | `Bitrix24Service::listSmartProcesses` |
| Права доступа к СП: user/department/all | Реализовано | `SmartProcessPermissionService`, `PermissionController` |
| Отдельная админка админов приложения | Реализовано | `AppAdminController`, `portal_app_admins` |
| Скачивание Excel-шаблона | Реализовано | `SmartProcessTemplateController`, `ExcelTemplateService` |
| Формат UF-кодов `ufCrm_*` | Реализовано | `BitrixFieldCode` |
| Импорт Excel построчно | Реализовано | `SmartProcessImportProcessor` |
| Batch-импорт 50+ записей | Реализовано | `Bitrix24Service::addItemsBatch`, `BITRIX24_BATCH_SIZE` |
| Прогресс импорта (processed/success/errors) | Реализовано | `ImportController::status`, `import_jobs` |
| Ошибки по строкам + файл ошибок | Реализовано | `import_job_errors`, `ExcelImportService::generateErrorReport` |
| Сводка после завершения | Реализовано | `resources/views/imports/show.blade.php` |
| MySQL для настроек/прав/логов | Реализовано | миграции `database/migrations/*` |
| Git flow `test -> main` | Реализовано процессно | ветки репозитория + release процесс |

## 2. Чек-лист ручного smoke-теста

1. Открыть приложение из меню Bitrix24.
Ожидаемо: в интерфейсе виден портал и `Пользователь: ID <не 0>`.

2. Список СП.
Ожидаемо: видны только разрешённые СП (или все, пока правила прав не настроены).

3. Скачивание шаблона.
Ожидаемо: файл `xlsx` скачивается, заголовки вида `Название (code)`, UF-коды в формате `ufCrm_*`.

4. Позитивный импорт (3-5 валидных строк).
Ожидаемо: `N успешно`, `0 ошибок`.

5. Негативный импорт (2 строки с ошибками).
Ожидаемо: строки не создаются, есть ошибки, доступна кнопка скачивания ошибок.

6. Скачивание файла ошибок.
Ожидаемо: скачивается `xlsx`, есть колонка `Ошибка`.

7. Batch-проверка (120+ строк).
Ожидаемо: импорт не падает, прогресс доходит до завершения.

8. Проверка прав.
Ожидаемо: у разрешённого пользователя СП виден, у неразрешённого скрыт.

## 3. Критерии “сдано”

- Пункты 1-8 smoke-теста пройдены.
- Нет `500/404` в ключевых сценариях (шаблон, импорт, файл ошибок).
- Очередь работает стабильно (`queue worker` активен).
- Есть бэкапы БД и `storage`.
- Мониторинг логов/очереди включён.

## 4. Известные ограничения MVP

- Не реализована верификация подписи входящих запросов Bitrix24.
- Сложные типы полей (например, `file`) намеренно исключены из импорта.
- Требуется постоянный `queue worker` в продакшене.
